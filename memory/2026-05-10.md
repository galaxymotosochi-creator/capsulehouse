## Что сделано за сессию

### Редизайн и логика кассы
- **PIN**: макс 4 цифры, анимация тряски при ошибке, мастер PIN 8888
- **Открытие смены**: экран с полем суммы и живым временем
- **Смена не закрывается** при нажатии "Назад" — остаётся открытой
- **Проверка открытой смены** при входе — если есть, сразу в кассу
- **Закрытие смены** через шестерёнку ⚙ со статистикой

### Дизайн верхней панели кассы
- Белый фон, разделители `|`, без серых плашек
- ◀ Стрелка (серая, без кнопки), 👤 Имя кассира, Все/Товары/Услуги
- 📷 🔍 ⚙ справа
- Размер шрифта 0.82rem (как кнопки "Карта")
- 🔢 ручной ввод — удалён
- Чек/Клиент кнопки — удалены (никуда не вели)
- Смайлы качающиеся (emoji-bounce)

### Карточки товаров в кассе
- Показывают остаток на складе 📦
- Если 0 — красная обводка, при клике тряска + тост
- Длинные названия обрезаются через `...`
- Шрифты и смайлы крупнее

### Нижняя часть кассы (левая панель)
- "К оплате:" вместо "Итого"
- [⋯] меню: Комментарий к чеку, Очистить чек
- [🖨] принтер (заглушка)
- Кнопка [💳 Оплатить]

### Модалка оплаты
- Клавиатура цифр слева (для наличных)
- "К оплате:" справа с суммой + "Внесено:"
- Способ оплаты: 💵 💳 📲
- 👤 Клиент — тумблер (при включении открывает выбор)
- ☑ Напечатать чек — тумблер
- 🔴 Закрыть без оплаты + 💳 Оплатить (компактные кнопки)

### Клиенты
- Поиск по имени/телефону
- Добавление нового (имя + телефон + комментарий)
- Поле "Комментарий" + кнопка "Добавить +"
- Необязательно — можно закрыть без клиента

### Тост
- Зелёное "Чек успешно добавлен!", само исчезает через 1.8 сек

### Баги, которые были и починены
1. **Синтаксическая ошибка** — пропущенная `}` в ACTION_MENU_ITEMS (ломала весь JS)
2. **Геар-меню** — не обновлялся HTML на Vercel (был старый CSS)
3. **selectClient** — не перерисовывал модалку подтверждения
4. **class mismatch** — HTML использовал reg-filter-btn, CSS ждал reg-tb-btn
5. **Утеряны функции** payR, addRI, upR, setPM, remRI, incRI, decRI (после редизайна)
6. **Высота кассы** — был просвет снизу, виден фон
7. **Плашка модальная** — не адаптировалась, обрезалось верхнее меню (откатили)
8. **Остаток товара** — двойной учёт (сейчас починен через getStock)

### Технические замечания
- Обновление на Vercel: git commit → push → ~30 сек на деплой
- Кэш браузера часто мешает — рекомендовать Ctrl+Shift+R
- Для проверки актуальной версии — режим инкогнито
